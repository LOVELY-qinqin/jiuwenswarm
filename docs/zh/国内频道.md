# 国内频道

JiuwenSwarm 支持接入多个国内聊天平台，以下是各频道的详细配置说明。

## 小艺

[演示视频](../assets/videos/xiaoyi_channel.mp4)

### 1. 创建小艺智能体

通过在 [小艺开放平台 ](https://developer.huawei.com/consumer/cn/hag/abilityportal/#/)创建 JiuwenSwarm 模式的智能体，即可与 JiuwenSwarm 服务无缝接入。

#### 第一步：创建 JiuwenSwarm 模式的智能体

![image-20260303094525671](../assets/images/小艺创建智能体.png)

#### 第二步：创建凭证及添加白名单

点击新建凭证，并​**保存 ak 与 sk 信息**​。

![image-20260303095143009](../assets/images/小艺创建凭证_1.png)

![image-20260303095210412](../assets/images/小艺创建凭证_2.png)

小艺开放平台为开发者提供真机调试能力，通过配置白名单分组，添加用户账号，可以在鸿蒙终端与小艺智能体对话。添加成功并上架成功后，可在终端小艺应用中使用该智能体。

![image-20260303100045878](../assets/images/小艺创建白名单_1.png)

勾选新建的用户组：

![image-20260303100252934](../assets/images/小艺创建白名单_4.png)

#### 第三步：上架智能体

填写开场白对话，点击上架按钮：

![image-20260303100240509](../assets/images/小艺智能体上架.png)

#### 第四步：启用 push 通知（可选）

点击开启触发器，填写名称后保存，将 apiID 填写至 jiuwenswarm 中：

![image-20260326201013711](../assets/images/xiaoyi_hook.png)

点击变量编辑，开通系统变量中的 push_id：

![image-20260326201211570](../assets/images/xiaoyi_push_id.png)

> 💡 **说明**：`api_id` 是触发器的 API 标识，`push_id` 是用于接收推送消息的标识。启用 push 通知时，两者需配合使用，缺一不可。

### 2. 绑定频道

#### 方式一：网页配置

将小艺开放平台获取到的 **ak、sk、agentId** 信息，填写至 JiuwenSwarm 的小艺频道，开启使用，点击保存即可开启对话。如开启了 push 可以填入 api_id（选填）：

![image-20260326201752051](../assets/images/小艺频道开启.png)

#### 方式二：修改配置文件

编辑 `~/.jiuwenswarm/config/config.yaml`：

``````yaml
channels:
  xiaoyi:
    mode: xiaoyi_channel
    ak: "小艺平台凭证 ak"
    sk: "小艺平台凭证 sk"
    agent_id: "创建的agent ID"
    api_id: "创建的触发器apiId"
    push_id: "触发器的 push_id（与 api_id 配合使用，启用 push 通知时必填）"
    uid: ""
    api_key: ""
    push_url: ""
    file_upload_url: ""
    phone_tools_enabled: false
    send_file_allowed: true
    enable_streaming: true
    enabled: true
``````

保存后若服务已运行会自动重载，未运行可以执行 `jiuwenswarm-start` 启动服务。

## 飞书

### 1. 创建飞书自建应用

1. 访问 [飞书开放平台](https://open.feishu.cn/) 并登录。

2. 进入**开发者后台**，点击**创建企业自建应用**。

3. 填写应用名称、描述，并上传图标，点击**创建**。

   ![image-20260312202028169](../assets/images/feishu.png)

### 2. 添加机器人能力

1. 在应用配置页面，左侧选择**添加应用能力**。

2. 点击**机器人**下的**添加**按钮。

   ![image-2026031220214646](../assets/images/feishu_add_robot.png)

### 3. 记录机器人应用凭证

1. 进入飞书机器人管理后台

2. 记录 App ID 与 App Secret 配置到 JiuwenSwarm 飞书频道，点击 enabled 并保存生效

   ![image-20260312202356784](../assets/images/feishu_app_token.png)

   ![image-20260312202651317](../assets/images/feishu_channel_config.png)

### 4. 配置权限

1. 左侧选择**权限管理** -> **API 权限**。

2. 搜索并开通以下关键权限（用于收发消息）：
   - `im:message:send`：以应用的身份发送消息
   - `im:message.p2p_msg:readonly`：获取用户发给机器人的单聊消息
   - `im:message.group_at_msg:readonly`：接收群聊中 @ 机器人消息事件
   - `im:resource:upload`：上传图片和文件（用于发送图片/文件给用户）
   - `contact:user.employee_id:readonly`：获取用户 ID 信息

![image-20260312203059819](../assets/images/feishu_app_permission.png)

### 5. 配置事件订阅（接收消息）

1. 左侧选择**事件与回调**

2. **添加事件**：
   - `im.message.receive_v1`（接收消息事件）
   - `im.message.message_read_v1`（消息已读）

3. **添加回调**：
   - `card.action.trigger`（卡片回传交互）

4. （可选）**配置加密策略**：如果启用了加密，需要保存 `Encrypt Key`

![image-20260312203514012](../assets/images/feishu_app_events.png)

### 6. 发布应用

1. 左侧选择**版本管理与发布**，点击**创建版本**

2. 填写版本号、更新说明，选择可用范围（通常选择**全体成员**或部分成员）

3. 提交审核。如果企业开启了免审，版本会立即生效

4. 飞书 APP 中登录提交应用的账号即可看到发布的聊天机器人

![image-20260312203730114](../assets/images/feishu_app_release.png)

### 7. 在群聊中添加机器人（可选）

1. 打开飞书客户端，进入需要添加机器人的群组

2. 点击群设置 -> **群机器人** -> **添加机器人**，搜索你创建的应用名称并添加

![image-20260312205356059](../assets/images/feishu_chat.png)

### 8. 配置飞书 Channel

启动前端服务后，在「频道-飞书」中打开 Enable 开关并配置第三步记录的 App ID 与 App Secret。

### 9. 开启群聊数字分身（可选）

在完成飞书机器人的基本配置后，可进一步开启数字分身功能，让机器人在群聊中作为指定用户的分身自动回复。

> 飞书中数字分身支持以下触发方式：**@机器人**、**@用户名**（即数字分身对应的用户）、或在消息中**提及用户姓名**，均会触发数字分身回复。

#### 前提条件

- 已完成飞书机器人创建、发布，并已将机器人添加到目标群聊（参考步骤 7）

#### 配置步骤

1. 在 JiuwenSwarm 频道管理页面，进入飞书频道配置，启用 **`group_digital_avatar`** 开关，并配置 **`my_user_id`** 和 **`bot_name`**。

   ![飞书数字分身开关](../assets/images/feishu_group_avatar.png)

2. 配置 **`my_user_id`**（必填）：填入数字分身对应用户的飞书 open_id。获取方式如下：
   - 登录飞书 API 调试台，进入[发送消息接口](https://open.feishu.cn/document/server-docs/im-v1/message/create)
   - 将 `receive_id_type` 选择为 **open_id**
   - 点击**快速复制 open_id**，在弹窗内选择目标用户，获取的 open_id 即为 `my_user_id`

   ![获取飞书open_id步骤1](../assets/images/feishu_user_id_1.png)

   ![获取飞书open_id步骤2](../assets/images/feishu_user_id_2.png)

3. 配置 **`bot_name`**：填写机器人在群聊中的显示名称，用于 @ 提及识别

4. （可选）启用 **`enable_memory`** 开关，开启后飞书群聊中可读取及搜索本地记忆文件

5. 配置**工具/路径权限**：数字分身在群聊中自主运行，无法像私聊那样弹窗让用户确认操作，因此需要预先配置允许使用的工具和可访问的路径。在频道管理页面开启数字分身后，进入权限配置界面，按需设置各工具的权限（`allow` 允许 / `deny` 拒绝）。未配置权限时，需要确认的操作（`ask`）会被自动降级为拒绝（`deny`）

   ![飞书数字分身权限配置](../assets/images/feishu_group_avatar_permission.png)

也可以通过编辑 `~/.jiuwenswarm/config/config.yaml` 来配置：

``````yaml
channels:
  feishu:
    app_id: "你的 App ID"
    app_secret: "你的 App Secret"
    enabled: true
    group_digital_avatar: true
    my_user_id: "ou_xxxx"
    bot_name: "机器人名称"
    enable_memory: false

permissions:
  owner_scopes:
    feishu:
      "ou_xxxx":
        defaults:
          "*": "allow"
        tools:
          bash:
            "*": "deny"
            patterns:
              "git status *": "allow"
              "git log *": "allow"
          write:
            "*": "deny"
  deny_guidance_message: "该工具未被授权在数字分身模式下使用。"
``````

6\. 飞书机器人需要新增以下权限：

   - `im:message.group_msg:readonly` - 获取群组中所有消息（敏感权限）
   - `contact:contact.base:readonly` - 获取通讯录基本信息
   - `contact:user.base:readonly` - 获取用户基本信息
   - `im:message.p2p_msg:readonly` - 获取用户发给机器人的单聊消息

#### 配置项说明

| 配置项 | 说明 |
|:------|:-----|
| `group_digital_avatar` | 是否启用群聊数字分身功能 |
| `my_user_id` | 开启数字分身时**必填**：数字分身对应用户的飞书 open_id |
| `bot_name` | 机器人在群聊中的名称，用于 @ 提及识别 |
| `enable_memory` | 是否启用群聊记忆功能 |
| `owner_scopes` | 数字分身工具权限配置，按 `channel_id` + `user_id` 维度控制各工具的权限 |

### 10. 飞书多 Bot（`feishu_enterprise`）

当一个团队需要在同一套 JiuwenSwarm 服务中接入多个飞书应用（多个 Bot）时，可使用 `channels.feishu_enterprise`。

每个 bot 会以独立 channel 运行，channel_id 格式为：`feishu_enterprise:<app_id>`。

当前多 bot 仅支持手动编辑 `~/.jiuwenswarm/config/config.yaml`：

``````yaml
channels:
  feishu_enterprise:
    bot_a:
      app_id: "cli_xxx"
      app_secret: "xxx"
      encrypt_key: ""
      verification_token: ""
      allow_from: []
      enable_streaming: true
      chat_id: ""
      enabled: true
    bot_b:
      app_id: "cli_yyy"
      app_secret: "yyy"
      encrypt_key: ""
      verification_token: ""
      allow_from: []
      enable_streaming: true
      chat_id: ""
      enabled: true
``````

#### 字段说明

| 配置项 | 说明 |
|:------|:-----|
| `app_id` | 飞书应用 App ID（必填） |
| `app_secret` | 飞书应用 App Secret（必填） |
| `encrypt_key` | 事件订阅加密 Key（可选） |
| `verification_token` | 事件订阅 Token（可选） |
| `allow_from` | 允许的用户 open_id 列表，空列表表示不限制 |
| `enable_streaming` | 是否启用流式下发 |
| `chat_id` | 固定推送目标（可选） |
| `enabled` | 是否启用该 bot |

#### 与单 Bot `feishu` 的区别

- `feishu`：单通道，channel_id 固定为 `feishu`
- `feishu_enterprise`：多通道，每个 bot 使用独立 channel_id（`feishu_enterprise:<app_id>`）
- 多 Bot 场景下，会按 bot 维度分别记录最近会话信息，避免 bot 之间互相覆盖

## 钉钉

### 1. 前提条件

- 你必须是**企业管理员**或**拥有开发者权限的成员**的帐号
- 企业已开通**钉钉开发者后台**

### 2. 创建企业内部应用机器人

#### 步骤 1：进入钉钉开发者后台

- 访问 [https://open-dev.dingtalk.com](https://open-dev.dingtalk.com)
- 登录后选择 **应用开发 → 企业内部开发 → 创建应用**

![image-20260313090856290](../assets/images/dingding_start.png)

#### 步骤 2：填写应用信息

- 应用名称：如 `JiuwenSwarm`
- 应用类型：**机器人**

![image-20260313091100590](../assets/images/dingding_create_app.png)

#### 步骤 3：添加机器人能力

- 创建成功后，进入应用详情页
- 点击 **应用能力 → 机器人 → 开启机器人配置**
- 填写机器人名称（如 `JiuwenSwarm`）、机器人简介（≤10 字）等信息

![image-20260313091410676](../assets/images/dingding_robot_config.png)

#### 步骤 4：配置消息接收模式

选择 **Stream 模式**，使用 WebSocket 长连接，无需公网 IP，适合云函数/本地调试。

### 3. 配置权限

在 **权限管理** 页面，开启以下权限（根据需求）：
- 发送单聊/群聊消息：`qyapi_robot_sendmsg`
- 根据手机号查 userId（用于单聊）：`topapi_v2_user_getbymobile`
- 发送互动卡片：`Card.Instance.Write`
- 流式更新卡片（AI 场景必备）：`Card.Streaming.Write`

### 4. 发布应用 & 机器人

#### 步骤 1：保存并发布机器人

- 在机器人配置页点击 **保存**
- 返回应用主页，点击 **版本管理与发布 → 发布**
- 填写版本号、更新日志、可见范围等信息

![image-20260313091956452](../assets/images/dingding_robot_publish.png)

#### 步骤 2：确认发布

- 点击 **确认发布**
- 状态变为 **已发布** 即可使用

> 💡 发布后可在钉钉客户端搜索机器人名称添加到群聊或私聊

### 5. 配置钉钉 Channel

从**凭证与基础信息**获取应用凭证 **Client ID** 和 **Client Secret** 。

**JiuwenSwarm**服务页面，在"频道-钉钉"中打开 Enabled 开关，并配置 client_id 与 client_secret 并保存：

![image-20260313092328435](../assets/images/dingding_channel_enable.png)

## 企业微信

### 1. 下载并安装企业微信客户端

1. 打开企业微信客户端，进入**工作台->智能机器人**，点击**创建机器人->手动创建**

    ![企业微信创建机器人入口](../assets/images/wecom/1_企业微信创建机器人入口.png)

    ![企业微信手动创建](../assets/images/wecom/2_创建机器人.png)

    ![企业微信创建页面](../assets/images/wecom/3_手动创建.png)

2. 进入创建页面后，选择**API 模式创建**

    ![企业微信API模式创建](../assets/images/wecom/4_API模式创建.png)

3. 在 API 配置页面，选择**连接方式为「使用长连接」**

    ![企业微信长连接配置](../assets/images/wecom/5_选择长连接.png)

### 4. 开启群聊数字分身（可选）

> ⚠️ **注意**：企业微信中，群聊消息必须 **@机器人** 才能被 bot 接收到。未 @ 机器人的消息不会触发数字分身回复。

#### 前提条件

- 已完成企业微信机器人创建并关联 JiuwenSwarm
- 已将机器人添加到目标群聊：打开企业微信，进入群组，点击添加成员 -> **群机器人** -> **智能机器人**，搜索你创建的应用名称并添加

#### 配置步骤

1. 在 JiuwenSwarm 频道管理页面，进入企业微信频道配置，启用 **`group_digital_avatar`** 开关，并配置 **`my_user_id`** 和 **`bot_name`**。启用数字分身前需要与机器人私聊一次，否则会报错 `[AiBotSDK] [WARN] Reply ack error`

   ![企业微信数字分身开关](../assets/images/wecom/14_group_avatar.png)

2. 配置 **`my_user_id`**（必填）：填入数字分身对应用户的企业微信账号。获取方式如下：
   - 打开[企业微信管理后台](https://work.weixin.qq.com/wework_admin/login)
   - 进入 **通讯录 -> 组织架构 -> 所在部门 -> 成员详情**
   - 页面中显示的**账号**即为 `my_user_id`

   ![获取企业微信user_id步骤1](../assets/images/wecom/12_user_id_获取.png)

   ![获取企业微信user_id步骤2](../assets/images/wecom/13_user_id_获取_2.png)

3. 配置 **`bot_name`**（可选）：填写机器人在群聊中的显示名称，用于 @ 提及识别

4. （可选）启用 **`enable_memory`** 开关，开启后企业微信群聊中可读取及搜索本地记忆文件（群聊中不写入记忆）

5. 配置**工具/路径权限**：数字分身在群聊中自主运行，无法像私聊那样弹窗让用户确认操作，因此需要预先配置允许使用的工具和可访问的路径。在频道管理页面开启数字分身后，进入权限配置界面，按需设置各工具的权限（`allow` 允许 / `deny` 拒绝）。未配置权限时，需要确认的操作（`ask`）会被自动降级为拒绝（`deny`）

   ![企业微信数字分身权限配置](../assets/images/wecom/15_group_avatar_permission.png)

也可以通过编辑 `~/.jiuwenswarm/config/config.yaml` 来配置：

``````yaml
channels:
  wecom:
    bot_id: "你的 Bot ID"
    secret: "你的 Secret"
    send_file_allowed: true
    enabled: true
    group_digital_avatar: true
    my_user_id: "用户账号"
    bot_name: "机器人名称"
    enable_memory: false

permissions:
  owner_scopes:
    wecom:
      "用户账号":
        defaults:
          "*": "allow"
        tools:
          bash:
            "*": "deny"
            patterns:
              "git status *": "allow"
              "git log *": "allow"
          write:
            "*": "deny"
  deny_guidance_message: "该工具未被授权在数字分身模式下使用。"
``````

#### 配置项说明

| 配置项 | 说明 |
|:------|:-----|
| `group_digital_avatar` | 是否启用群聊数字分身功能 |
| `my_user_id` | 开启数字分身时**必填**：数字分身对应用户的企业微信账号 |
| `bot_name` | 可选：机器人在群聊中的名称，用于 @ 提及识别 |
| `enable_memory` | 是否启用群聊记忆功能 |
| `owner_scopes` | 数字分身工具权限配置，按 `channel_id` + `user_id` 维度控制各工具的权限 |

## 个人微信

### 1. 前提条件

- 你是**安卓**或 **iOS** 用户
- 你是**鸿蒙**用户，且不介意使用**卓易通**
- 由于当前个人微信限制，长时间未在个人微信中对话，定时任务无法发出，建议采用其他通道设置定时任务

> 💡 **提示**：当用户向 ClawBot 发送一条消息后，应用最多可以向该用户发送 **10 条**独立消息（包括回复的消息）。若用户再次向 ClawBot 发送消息，配额将重置为 10。

### 2. 安卓或 iOS 接入

#### 步骤 1：微信版本升级

在微信中依次进入「我」->「设置」->「关于微信」->「版本更新」：
- iOS 请升级到最新版本
- 安卓请升级到最新版本

![wechat_update](../assets/images/wechat_update.png)

#### 步骤 2：微信扫码接入

- 打开最新版本的 JiuwenSwarm 前端，点击「频道」->「微信」，启用微信配置功能并点击保存。页面会出现**二维码**

![jiuwenswarm_enable_wechat](../assets/images/jiuwenswarm_enable_wechat.png)

- 打开手机微信，通过右上角「+」->「扫一扫」，扫描上一步骤生成的二维码，继续连接即可完成接入

### 3. 鸿蒙接入

由于鸿蒙微信版本还未支持 **ClawBot** 功能，所以**鸿蒙原生微信当前还不能接入**，但用户可以通过**卓易通**完成接入，步骤如下：

#### 步骤 1：安装微分身双开

- 通过「卓易通」下载「微分身双开」，重新登录微信

![wechat_harmony](../assets/images/wechat_harmony.png)

#### 步骤 2：升级 & 接入

- 其余步骤与**安卓或 iOS 接入**方法完全一致。
