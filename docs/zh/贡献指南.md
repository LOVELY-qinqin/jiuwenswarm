## 如何贡献

### 提交 Issue

在 [GitCode Issues](https://gitcode.com/openJiuwen/jiuwenswarm/issues) 中提交，请选择合适的类型：

| 类型 | 模板 | 说明 |
|------|------|------|
| Bug 报告 | Bug Report | 包含复现步骤、预期行为、实际行为、环境信息 |
| 功能建议 | Feature Request | 描述场景、期望效果、替代方案 |
| 使用问题 | Question | 描述遇到的问题、已尝试的排查方式 |

**Bug 报告建议包含：**

1. JiuwenSwarm 版本号（`jiuwenswarm --version`）
2. 操作系统与 Python 版本
3. 复现步骤
4. 预期行为 vs 实际行为
5. 相关日志（`~/.jiuwenswarm/logs/` 和 `~/.jiuwenswarm/agent/.logs`）

### 提交 Pull Request

1. **Fork 仓库**：在 GitCode 上 Fork [openJiuwen/jiuwenswarm](https://gitcode.com/openJiuwen/jiuwenswarm)

2. **创建分支**：

```bash
git checkout -b feature/your-feature-name
# 或
git checkout -b fix/your-bug-fix
```

3. **开发与测试**：修改代码后运行测试

```bash
uv run pytest tests/unit_tests/
```

4. **提交代码**：

```bash
git add .
git commit -m "feat: 添加 XXX 功能"
```

5. **推送并创建 PR**：

```bash
git push origin feature/your-feature-name
```

在 GitCode 上创建 Pull Request，目标分支为 `develop`。

**PR 描述建议包含：**

- 变更内容与动机
- 关联的 Issue（如 `Closes #123`）
- 测试方式
- 截图或录屏（如涉及 UI 变更）

### 分享 Skill

将你开发的 Skill 发布到 [Swarm Skills Hub](https://swarmskills.openjiuwen.com/)，让更多开发者搜索、安装和复用。

## 开发环境搭建

详细的环境搭建指南请参考 [开发者文档](developer_guide.md)，核心步骤：

```bash
# 克隆仓库
git clone https://gitcode.com/openJiuwen/jiuwenswarm.git
cd jiuwenswarm

# 创建虚拟环境
uv venv --python=3.11
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# 安装依赖
uv sync

# 初始化并启动
uv run jiuwenswarm-init
uv run jiuwenswarm-start
```

## 代码规范

### 代码风格

- **Python**：遵循 [PEP 8](https://peps.python.org/pep-0008/)，使用类型注解
- **TypeScript / React**：遵循项目 ESLint 配置
- **文档**：中英文文档保持同步更新

### 提交信息规范

采用 [Conventional Commits](https://www.conventionalcommits.org/) 格式：

```
<type>(<scope>): <description>
```

| type | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 文档变更 |
| `style` | 代码格式（不影响功能） |
| `refactor` | 重构（非新功能、非修复） |
| `test` | 测试用例 |
| `chore` | 构建、依赖、工具变更 |

示例：

```
feat(swarm): 添加 Swarmflow 有状态算子支持
fix(config): 修复模型配置保存后未自动重启的问题
docs: 更新安装指南中的 Python 版本要求
```

### 分支命名规范

| 前缀 | 用途 | 示例 |
|------|------|------|
| `feature/` | 新功能 | `feature/swarmflow-stateful-operator` |
| `fix/` | Bug 修复 | `fix/config-restart-issue` |
| `docs/` | 文档更新 | `docs/update-install-guide` |
| `refactor/` | 代码重构 | `refactor/llm-client` |

## PR 审查流程

1. **自动检查**：PR 创建后自动运行测试
2. **代码审查**：至少两位 Committer 审查通过
3. **修改反馈**：根据审查意见修改后重新推送
4. **自动合入**：两位审查通过后，由 openJiuwen-bot 自动合入 `develop`

**审查关注点：**

- 代码逻辑正确性
- 是否有对应的测试用例
- 是否影响现有功能
- 文档是否同步更新

## 版本发布

- 开发分支：`develop`
- 发布分支：从 `develop` 创建 `release/vX.Y.Z`
- 版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)：`MAJOR.MINOR.PATCH`
- 发布后在 [GitCode Releases](https://gitcode.com/openJiuwen/jiuwenswarm/releases) 创建 Release Notes
