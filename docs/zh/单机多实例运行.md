# 单机多实例运行

支持在同一台机器上运行多个独立的 JiuwenSwarm 实例，每个实例拥有独立的工作区、配置和端口。

## 适用场景

- **开发与生产隔离**：在开发机上同时运行测试实例和生产实例
- **多租户环境**：为不同用户或项目创建独立的 Agent 实例
- **多模式并行**：同时运行不同配置的 Agent（如不同模型、不同权限策略）

> **与「多窗口 TUI」的区别**：本页描述的是**多个独立后端实例**（不同工作区、不同端口）。若只需在同一 Gateway 上开多个 TUI 终端窗口，见 [TUI 使用指南 — 多窗口 TUI](TUI使用指南.md#多窗口-tui)，无需创建额外实例。

---

## 核心概念

### 实例隔离

每个实例具有独立的：

| 资源 | 说明 |
|------|------|
| **工作区目录** | `workspace/` 目录，包含 SOUL.md、skills、memory 等 |
| **配置文件** | `.env` 文件，包含 API_KEY、模型配置等 |
| **端口** | AgentServer、Gateway、WebChannel、Frontend 各端口独立 |
| **进程** | 独立进程组，通过 PID 文件管理 |
| **启动锁** | 防止同一实例被多个终端同时启动 |

### 实例命名规则

- 长度：1-64 字符
- 允许字符：字母、数字、下划线、连字符
- 禁止以 `.` 开头
- 保留名称：`default`、`config`、`tmp`、`jiuwenswarm`、`all`

## 配置文件

### instances.yaml

位置：`~/.jiuwenswarm/instances.yaml`（或仓库根目录）

```yaml
instances:
  dev:
    workspace: ~/.jiuwenswarm_dev
    ports:
      agent_server: 19092
      web: 20000
      gateway: 20001
      frontend: 6173
  prod:
    workspace: ~/.jiuwenswarm_prod
    ports:
      agent_server: 20092
      web: 21000
      gateway: 21001
      frontend: 7173
```

### 端口自动分配算法

默认端口 = 基准端口 + 实例索引 × 1000

| 服务类型 | 基准端口 | 默认实例 (index=0) | 第一个命名实例 (index=1) |
|----------|----------|-------------------|-------------------------|
| agent_server | 18092 | 18092 | 19092 |
| web | 19000 | 19000 | 20000 |
| gateway | 19001 | 19001 | 20001 |
| frontend | 5173 | 5173 | 6173 |

## 命令使用

### jiuwenswarm-init --name

创建命名实例：

```bash
# 创建 dev 实例
jiuwenswarm-init --name dev

# 创建 prod 实例（指定工作区）
jiuwenswarm-init --name prod --workspace ~/.jiuwenswarm_prod
```

执行后会：
1. 创建工作区目录
2. 生成实例专属 `.env` 文件
3. 更新 `instances.yaml` 配置
4. 分配端口（自动或手动指定）

### jiuwenswarm-start 管理命令

```bash
# 列出所有实例状态
jiuwenswarm-start --list

# 输出示例：
# INSTANCE     STATUS     PID     WORKSPACE                               PORTS
# --------------------------------------------------------------------------------
# default      running    12345   ~/.jiuwenswarm                           as:18092,w:19000,g:19001,f:5173
# dev          stopped    -       ~/.jiuwenswarm_dev                       as:19092,w:20000,g:20001,f:6173
# prod         stopped    -       ~/.jiuwenswarm_prod                      as:20092,w:21000,g:21001,f:7173

# 查看指定实例详情
jiuwenswarm-start --status dev

# 启动命名实例
jiuwenswarm-start --name dev
jiuwenswarm-start --name dev app    # 仅启动后端
jiuwenswarm-start --name dev web    # 仅启动 Web 服务

# 停止实例
jiuwenswarm-start --stop dev

# 重启实例
jiuwenswarm-start --restart dev
jiuwenswarm-start --restart dev app --mode app
```

## 启动锁机制

为防止同一实例被多个终端同时启动，系统使用文件锁：

- 锁文件：`<workspace>/.instance.lock`
- 锁超时：30 秒（`STALE_LOCK_TIMEOUT`）
- 跨平台支持：
  - Unix：`fcntl.flock`
  - Windows：独占文件创建

若启动时遇到锁冲突：

```
[start_services] ERROR: Instance 'dev' startup in progress by another process
[start_services] Wait a few seconds or check if another terminal is starting this instance.
```

## 环境变量

| 变量 | 说明 |
|------|------|
| `JIUWENSWARM_DATA_DIR` | 覆盖数据根目录（影响 instances.yaml 位置） |
| `JIUWENSWARM_CONFIG_DIR` | 覆盖配置目录 |

启动时可通过 `--dotenv` 指定实例专属配置：

```bash
# 内部机制：启动命名实例时自动加载 bootstrap .env
jiuwenswarm-start --name dev
# 等价于加载 ~/.jiuwenswarm_dev/.env
```

## PID 文件

每个实例在工作区目录维护 PID 文件：

- 文件名：`.instance.pid`
- 内容：进程 ID + 启动时间戳
- 用途：
  - 状态查询（`--status`、`--list`）
  - 进程控制（`--stop`、`--restart`）

## 最佳实践

### 1. 端口规划

避免端口冲突：

```bash
# 检查端口占用
jiuwenswarm-start --status dev
# 系统会自动检测并报告冲突
```

### 2. 配置隔离

每个实例使用独立的 API Key 或模型配置：

```bash
# dev 实例 .env
API_KEY="sk-dev-key"
MODEL_NAME="deepseek-chat"

# prod 实例 .env
API_KEY="sk-prod-key"
MODEL_NAME="gpt-4"
```

### 3. 工作区管理

实例工作区包含 Agent 所有运行数据：

```
~/.jiuwenswarm_dev/
├── .env                # 实例配置
├── .instance.pid       # 进程管理
├── .instance.lock      # 启动锁
├── workspace/
│   ├── agent/
│   │   ├── SOUL.md     # Agent 个性
│   │   ├── memory/     # 记忆存储
│   │   └── skills/     # 技能目录
│   └── session/        # 会话数据
```

## 常见问题

### Q: 如何删除实例？

手动删除工作区目录和 instances.yaml 条目：

```bash
rm -rf ~/.jiuwenswarm_dev
# 编辑 ~/.jiuwenswarm/instances.yaml 移除对应条目
```

### Q: 默认实例与命名实例的区别？

- **默认实例**：无 `--name` 参数启动，使用 `~/.jiuwenswarm/` 工作区
- **命名实例**：通过 `--name` 指定，使用独立工作区和端口

### Q: 多实例能否共享配置？

可以。在实例 `.env` 中引用共享配置：

```bash
# ~/.jiuwenswarm_dev/.env
source ~/.jiuwenswarm/.env  # 共享基础配置
MODEL_NAME="special-model" # 实例专属覆盖
```
