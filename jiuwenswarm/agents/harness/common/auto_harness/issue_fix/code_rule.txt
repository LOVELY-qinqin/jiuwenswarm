2.1.2 命名注意事项
G.NAM.02 命名要有明确含义，以提升代码的可读性
G.NAM.03“self”应是实例方法的第一个参数，“cls”应是类方法的第一个参数
G.NAM.04 避免在无关的标识符或无关的概念之间重用名字， 避免因重名而导致的意外赋

2.2 注释
2.2.1 文档字符串
G.CMT.01 模块文档字符串写在文件的顶部，Shebang和文件编码声明之后，导入(imports)
部分之前的位置，不需要缩进
G.CMT.02 类的文档字符串写在类声明(class ClassName:)所在行的下一行，并向后缩进4
个空格
G.CMT.03 公共函数的文档字符串写在函数声明(def function_name(param):)所在行的下
一行，并向后缩进4个空格
2.2.2 代码注释
P.01 注释跟代码一样重要，应按需注释
G.CMT.04 代码的注释位置和格式应该在项目内保持统一
G.CMT.05 正式交付给客户的代码不应包含TODO/FIXME注释
G.CMT.06 文件头注释应该包含版权许可信息
